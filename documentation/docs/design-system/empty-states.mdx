# Empty States

Empty states appear when there's no content to display — a search returns nothing, a pattern list is empty, or a section hasn't been populated yet. A good empty state reduces confusion by explaining _why_ content is missing and offering a clear next step.

---

## Anatomy

Every empty state is built from four optional slots:

| Slot      | Required | Purpose                                              |
| --------- | -------- | ---------------------------------------------------- |
| `icon`    | No       | Visual anchor — emoji or SVG                         |
| `title`   | Yes      | Short, plain-language heading                        |
| `body`    | No       | One sentence explaining the situation and what to do |
| `actions` | No       | One or two CTAs — primary action first               |

## Scenario: Search — No Results

Use when a search query returns zero matches.

<EmptyState
  icon="🔍"
  title='No results for "timelock"'
  body='Double-check your spelling, or try broader terms like "escrow" or "token".'
  actions={[
    { label: 'Browse all patterns', href: '/docs/patterns/overview' },
    { label: 'Clear search', variant: 'secondary', href: '#' },
  ]}
/>

```tsx
// Typically rendered inside your search results component
{
  results.length === 0 && (
    <EmptyState
      icon="🔍"
      title={`No results for "${query}"`}
      body='Double-check your spelling, or try broader terms like "escrow" or "token".'
      actions={[
        { label: 'Browse all patterns', href: '/docs/patterns/overview' },
        { label: 'Clear search', variant: 'secondary', onClick: clearSearch },
      ]}
    />
  );
}
```

## Writing Guidelines

**Title** — state the situation plainly. Avoid "Oops" or "Uh oh".

| ✅ Do                         | ❌ Don't                 |
| ----------------------------- | ------------------------ |
| "No results found"            | "Oops! Nothing here."    |
| "No DeFi patterns yet"        | "This section is empty." |
| "Nothing matches your search" | "Error: no data."        |

**Body** — one sentence. Explain why and what to do next.

**Actions** — maximum two. Primary action first, secondary (or ghost) second.

## Props

| Prop        | Type                   | Default | Description               |
| ----------- | ---------------------- | ------- | ------------------------- |
| `title`     | `string`               | —       | Required. Short heading.  |
| `icon`      | `ReactNode`            | —       | Emoji, SVG, or component. |
| `body`      | `ReactNode`            | —       | Supporting copy.          |
| `actions`   | `EmptyStateAction[]`   | `[]`    | CTA buttons.              |
| `size`      | `'sm' \| 'md' \| 'lg'` | `'md'`  | Size variant.             |
| `className` | `string`               | —       | Extra CSS class.          |

### EmptyStateAction

| Prop      | Type                                  | Default                           | Description                         |
| --------- | ------------------------------------- | --------------------------------- | ----------------------------------- |
| `label`   | `string`                              | —                                 | Button text.                        |
| `href`    | `string`                              | —                                 | Link target (renders `<a>`).        |
| `onClick` | `() => void`                          | —                                 | Click handler (renders `<button>`). |
| `variant` | `'primary' \| 'secondary' \| 'ghost'` | First=`primary`, rest=`secondary` | Visual style.                       |
